Consensus Summary

All three models reached strong agreement that the platform cannot be built as designed due to critical security, legal, and feasibility flaws. Claude Opus initially framed this as "approve with major revisions" but shifted to "reject" in round 3 after cross-pollination revealed additional issues. The distinction between "revise" and "reject" is semantic — the severity of required changes is identical.

Dangerous Issues — Must Fix Before Building

#1 TCPA Compliance is Fatally Flawed

The FCC's 2024 ruling classifies AI voices as "artificial," requiring prior express written consent for all outbound calls to wireless numbers — not just marketing calls. The design wrongly treats tour reminders as "informational" needing only express consent.

All 3 models flagged this. Statutory damages: $500–$1,500 per call.

#2 Timezone Compliance Uses Wrong Timezone New

The pre-call gate checks calling hours based on the property's timezone, not the recipient's. TCPA restrictions apply based on the recipient's local time. The system lacks any mechanism to determine the lead's timezone (e.g., area code lookup), guaranteeing systematic violations.

Surfaced by Gemini, confirmed by all.

#3 RLS + Connection Pooling = Cross-Tenant Data Leak

Using session-level SET app.current_tenant_id with a connection pool allows connections to be recycled with the wrong tenant's context. This is the #1 multi-tenant security bug.

Fix: Use SET LOCAL within transactions — it auto-resets when the transaction ends.

#4 Platform-Wide API Keys are Catastrophic New

The explicit design of "one key per service for entire platform" means a single leaked Retell or Claude API key exposes every tenant's data and operations. Per-tenant or securely proxied credentials are required.

All 3 models flagged this independently.

#5 recording_consent DEFAULT true is Legally Reckless New

Any bug that bypasses consent-checking logic would record without consent. The database default must be false, not true.

#6 SSRF in Onboarding Scraper New

The onboarding wizard scrapes arbitrary user-provided URLs with no protection against internal IPs or cloud metadata endpoints. Classic Server-Side Request Forgery vector.

Fix: Strict URL validation + sandboxed execution environment.

#7 BIPA Risk for Illinois Residents New

If ElevenLabs or Retell stores voiceprints, Illinois residents require explicit BIPA consent. Statutory damages: $5,000 per violation. The design does not mention BIPA at all.

Significant Gaps

#8 Single-Process Express Bottleneck New

Webhooks (10s SLA), BullMQ workers (CPU-heavy transcript analysis), and cron scheduler all run in one process. A slow Claude API call starves webhook responses, causing Retell tool call timeouts.

Fix: Separate webhook server from workers as distinct Railway services.

#9 Missing LLM API Costs in Estimates New

The $800–850/mo cost estimate completely omits Claude API costs for transcript analysis, KB generation, gap detection, and onboarding scraping. Could add hundreds/month, further eroding thin margins.

#10 Timeline is ~2x Underestimated

All three models independently estimated 16–22+ weeks for a solo developer. No pre-existing code exists to reuse. Phase 1 alone (15 tasks) is a 6–8 week project.

#11 Single-Vendor Lock on Retell

Voice, transcription, widget, phone numbers, call routing — all through Retell. No abstraction layer, no fallback provider.

#12 Thin Margins (~43–47% Gross)

Starter $99/mo with 500 minutes: ~47% gross margin. Pro $299/mo with 2,000 minutes: ~43%. SaaS viability typically requires 70–80%. Recommend $199/$499 pricing.

#13 No Human Call Transfer

Design mentions "warm handoff" but no implementation — no Retell transfer_call API, no forwarding number config.

#14 No Embedding Fallback

OpenAI embedding API down = every search_knowledge call fails = every caller gets "I can't look that up." Add PostgreSQL tsvector full-text search as fallback.

What's Done Well

  • Multi-tenant with RLS from day 1 — correct decision, hard to retrofit
  • Three-layer knowledge engine — live data + curated KB + learned knowledge is a genuine competitive moat
  • Learning loop as differentiator — the compound advantage of getting smarter from every call is the real product
  • TCPA awareness — 7-step gate, consent audit trail, DNC checks show serious compliance intent (even if details need legal review)
  • Staleness detection — time-based + contradiction + site sync is sophisticated
  • Verified tech specs — Retell docs, pgvector benchmarks, Stripe metering all cross-checked against real documentation
  • Graceful degradation — fallback prompts when tools fail, agent instructions for mid-call error handling
  • Cost estimates with real numbers — most plans skip this entirely
  • Phased plan with clear boundaries — dependencies between phases are well-defined

Final Recommendation

Council's Prescribed Path Forward

  1. Do not build the outbound calling system before engaging a TCPA-specialized attorney. The legal exposure is existential and the design contains at least two systematic violation vectors (consent classification and timezone logic).
  2. Fix the RLS pooling bug — use SET LOCAL within transactions before writing any tenant-facing code.
  3. Redesign the API key architecture — per-tenant or securely proxied credentials, not platform-wide keys.
  4. Fix recording_consent DEFAULT true — change to false.
  5. Secure the scraper against SSRF before the onboarding wizard goes live.
  6. Separate webhook server from workers into distinct services.
  7. Target a realistic 10-week inbound-only MVP for 1–2 properties, deferring outbound, SaaS billing, and the marketing site to a second phase with a corrected legal and security foundation.
  8. Price higher ($199/$499) to achieve viable margins.
