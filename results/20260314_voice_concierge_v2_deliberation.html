New Issues Found by Council Not in v1

Critical Webhook Non-Idempotency

Retell retries webhooks up to 3x. The call_ended handler creates usage records and queues analysis without dedup checks. Retries cause duplicate billing and redundant Claude API calls.

Fix: Add UNIQUE constraint on retell_call_id, check existence before insert.

High Missing OPENAI_API_KEY in .env.example

Knowledge service uses OpenAI embeddings but the env template doesn't list the key. Deployments fail silently on first KB ingestion.

High Missing @aws-sdk/s3-request-presigner in package.json

Storage service imports getSignedUrl from this package but it's not in dependencies. Build fails on signed URL generation.

Medium No Worker Graceful Shutdown

Railway restarts kill in-progress BullMQ jobs (e.g., 30s transcript analysis) without waiting. Add SIGTERM handler with worker.close().

Medium No Input Validation on Admin Routes

req.body passed directly to DB queries despite Zod being in package.json. Invalid timezone strings or malformed settings cause downstream bugs.

Medium No Worker Health Check

API server has /health but worker service doesn't. Railway can't detect if workers are alive and connected to Redis.

Low Tool Call Endpoints Bypass RLS

Admin routes use RLS via SET LOCAL, but tool call endpoints use manual WHERE tenant_id = $1. Two access patterns = increased risk of a developer forgetting the manual filter on new endpoints.

Low Transaction Overhead on Read-Only Routes

The RLS fix wraps every request (including GETs) in BEGIN/COMMIT. Adds unnecessary overhead for read-heavy operations.

Must-Fix Before Building (Council Consensus)

  1. Redesign API key strategy — Per-tenant keys or proxy layer for Retell at minimum. This is the #1 unresolved security flaw across both deliberations.
  2. Webhook idempotency — UNIQUE constraint on retell_call_id, dedup in BullMQ. Prevents duplicate billing on retries.
  3. TCPA consent flexibility — Don't hardcode informational/marketing split. Either label all AI outbound as requiring written consent, or mark as pending attorney determination with default-to-written code path.
  4. Missing deps & env — Add OPENAI_API_KEY to .env.example, @aws-sdk/s3-request-presigner to package.json.
  5. Input validation — Zod schemas on admin routes (slug, timezone, settings).

Should-Fix Before Launch (Not Blocking Build)

  • Realistic timeline acknowledgment (16-20 weeks or scope cut to inbound-only MVP)
  • Worker graceful shutdown + health check endpoint
  • BIPA assessment for Illinois properties (confirm Retell doesn't store voiceprints)
  • Demo rate limiting and cost controls
  • Connection pool sizing documentation
