Grok's single multi-tenant app proposal is wrong and I want to address it definitively so this doesn't keep coming up:

Grok scored this 4/10 and 5/10 across two rounds largely on the basis that a single multi-tenant app would be "fundamentally better." This is incorrect for a platform where SEO is the primary revenue channel and independent domain authority is the product's competitive moat.

However: The plan's token migration strategy is where I break from Opus's Round 2 analysis. The critical finding:

Both repos already share the cx-* naming convention. LA has --color-cx-blue: var(--blue) where --blue: #2D4A3E (sage green). Lofts has --color-cx-blue: var(--blue) where --blue: #B87333 (copper). The existing system IS the design token system. Components use bg-cx-blue and get the brand's color.

My recommendation differs from both prior reviewers: Don't keep cx-* as-is AND don't do the full semantic rename. Instead:

This preserves backwards compatibility while making the token contract explicit.

Risk 3: current_setting('app.current_domain') will not work with Supabase's connection pooling.

Both Opus rounds identified this. I want to be more specific:

Supabase uses PgBouncer in transaction mode by default. The SET command used by current_setting() is a session-level command. In transaction mode, session state is NOT preserved between transactions. This means:

  • Transaction 1: SET app.current_domain = 'lofts' → queries return Lofts data ✓
  • Transaction 2 (same PgBouncer connection): app.current_domain may still be 'lofts' from the previous user's session, or undefined

This is a data isolation bug. A user on LA could see Lofts-only listings, or vice versa.

Mitigation: Don't use RLS session variables for domain isolation. The createDomainClient factory already adds application-layer domain filtering (.contains('domain_visibility', [domain])). That's the real gate. Keep RLS only for anon/authenticated access using JWT claims (which DO survive transaction-mode pooling). For service-role queries (ISR), the factory's filter is sufficient.
